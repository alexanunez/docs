1. Navigate to the Agent you are ready to deploy.
2. Verify the Agent is showing the correct **“Active Version”** from your recent testing.

   a. If an update is required, go into the Agent, select the version number in the top left and click **“Set Active”** then return to the list of Agents.
3. If the correct version is active, select the option menu on your Agent by clicking the **ellipsis (...)** to the right of your Agent.
4. Select **“Connection Info”** from the menu and you will be given a window to detailing the Agent’s unique identifiers and providing you with an example Agent Execution API call.
5. Airia provdes two examples API calls using Curl and Python scripts. Select the preferred examples using the drop down menu in the top right of the window.
6. Take this information and configure the API details within your chosen system, following their guidance and documentation.
7. If you have not yet generated an Airia API Key, this can be done in your platform under settings.

   a. Navigate to Airia API Keys, select **“New API Key”** in the top right.

   b. Enter the API Key Name, define the Scope of this key to the appropriate project then click **“Create”** to generate your Airia API Key.

In deploying your Agent via API, testing will need to be completed in your chosen tool which will call the Agent into action to complete the query.
